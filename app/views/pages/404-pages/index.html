404 pages

404 pages tell users that the page they were looking for As with all error messages the aim of a 404

Standard page

Use this page when the page that the user is looking for is not there.

Use this page when the page that the user is looking for is not there and there is an alternative support channel.

Broken link

Use this page when a user has clicked a link in the service and the page they requested is not there.

Use this page when a user has clicked a link in the service and the page they requested is not there and the service has an alternative channel they can use to complete their transaction.

Repeated attempts to access a page that's not there

Use this page when a user has clicked a link in the service and the page they requested is not there and the service has an alternative support channel that they can use to complete their journey.

Notes

  • this is a content design problem
  • there are two senarios either the user made a mistake or HMRC made a mistake

  • if we made a mistake we can:

    • tell them that their work is safe
    • help them to get back to a place where they can continue from
    • tell them that we are trying to resolve the broken link
    • point them to an alternative route if there is one avalible; such as phone support

  • If they made a mistake we can:

    • ask them to check the link
    • help them to get to a place where they were trying to get to We can't rely on telling them that they typed the link incorrectly as they may have cut and pasted the link from an email
